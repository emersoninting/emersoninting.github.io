VLAN Concepts & Configuration

Understanding VLAN Concepts

A structured guide on VLAN configurations, troubleshooting, and inter-VLAN routing.

Device Configurations

Manage SW1 Configuration:

(config)# vlan 10
(config-vlan)# name IT
(config-vlan)# exit
(config)# int vlan 10
(config-if)# ip add 10.10.10.10 255.255.255.0
(config-if)# description IT-SW1
(config-if)# int eth0/0
(config-if)# switchport mode access
(config-if)# description IT-ws1
(config-if)# switchport access vlan 10
(config-if)# int range eth0/1-2
(config-if-range)# shutdown
(config-if-range)# int eth0/3
(config-if)# switchport mode trunk

Note: If you encounter the error: "Command rejected: An interface whose trunk encapsulation is 'Auto' cannot be configured to 'trunk' mode", you need to set encapsulation.

(config-if)# switchport trunk encapsulation dot1q
(config-if)# switchport mode trunk
(config-if)# switchport trunk allowed vlan 10,20

Manage SW2 Configuration:

(config)# vlan 20
(config-vlan)# name HR
(config-vlan)# exit
(config)# int vlan 20
(config-if)# ip add 20.20.20.20 255.255.255.0
(config-if)# description HR-SW1
(config-if)# int eth0/0
(config-if)# switchport mode access
(config-if)# switchport access vlan 20

Situations & Troubleshooting

Situation 1: PC1 Cannot Connect to PC2

This is expected because PC1 is in VLAN 10 and PC2 is in VLAN 20, which means they are segmented. This is a key concept of VLAN segmentation.

Solution 1: Use the Same VLAN on Both Switches

Adding a new workstation to SW2 in VLAN 10 allows communication.

(config)# vlan 10
(config-vlan)# name IT
(config-vlan)# exit
(config)# int vlan 10
(config-if)# ip address 10.10.10.11 255.255.255.0
(config-if)# description IT-SW2
(config-if)# int eth0/1
(config-if)# switchport mode access
(config-if)# switchport access vlan 10

Solution 2: Inter-VLAN Routing (Router on a Stick)

Since switches are Layer 2 devices, they cannot route traffic between VLANs. To enable VLAN 10 and VLAN 20 communication, we use a router-on-a-stick configuration.

(config)# interface gig0/1
(config-if)# no shutdown
(config-if)# encapsulation dot1q 10
(config-if)# ip address 10.10.10.1 255.255.255.0
(config)# interface gig0/2
(config-if)# encapsulation dot1q 20
(config-if)# ip address 20.20.20.1 255.255.255.0
